Knitting is a shared-memory worker pool for Node.js, Deno, and Bun. You define
one or many tasks at module scope, export them, and then create a pool that can call
those tasks from the main thread.

## Core ideas
- **Speed**: Being between 10 to 100 times faster than other IPC options, 
  increases the range of functions that can be paralelized without the overhead
  eats the perfomance
- **Easy to use**: More than 30+ examples, fully typed, alined with JS conventions, well documented and easy to integrate exisiting projects.
- **Multi domain**: Created for the web and for math and simulation.




## Typical flow

1. Define tasks at module scope and export them.
2. Create a pool with `createPool({ threads })({ tasks })`.
3. Call tasks through `call.*()` or `fastCall.*()` and optionally `send()`.
4. Shut down workers when finished.
